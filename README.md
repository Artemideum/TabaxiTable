# TabaxiTable

Онлайн-стол для классической D&D 5e (правила 2014 года): комнаты по коду, синхронные интерактивные листы персонажей, общий боевой стол и броски костей. Версия приложения: **1.9.0**. Схема листа персонажа: **8**.

## Запуск на Windows

1. Установи [Node.js](https://nodejs.org/) версии 20 или новее.
2. Открой папку проекта в Проводнике.
3. Кликни по адресной строке папки, введи `cmd` и нажми Enter.
4. Выполни `npm install`.
5. Выполни `npm start`.
6. Открой `http://localhost:31777`. После первого входа TabaxiTable сам вернёт тебя в комнату при обновлении страницы.

## Обновление одной кнопкой

Если проект был один раз получен через GitHub (`git clone` или GitHub Desktop), запускай `update.bat`. Он подтянет только изменившиеся файлы и обновит зависимости. Комнаты из `data/rooms.json` и локальные настройки не отправляются в GitHub и не затираются обновлением.

Для игры через интернет проект нужно разместить на сервере. Пока сайт запущен только на компьютере, зайти на него сможет лишь хозяин компьютера и, при дополнительной настройке сети, люди из той же локальной сети.

## Что важно знать

- Просто запускай `start.bat`: он сам перейдёт в папку проекта, установит зависимости при первом запуске и откроет браузер.
- Не запускай второй экземпляр одновременно. Если порт занят, TabaxiTable попробует следующий свободный порт и напишет точный адрес в окне.
- Кнопка с логотипом выходит из комнаты и забывает автовход. Обычное обновление страницы, наоборот, возвращает в текущую комнату.
- Справочник содержит краткие самостоятельные описания 120 заклинаний; хоумбрю по-прежнему можно добавить вручную.
- `data/rooms.json` — твои живые комнаты, а `data/assets/` — загруженные карты и токены. Не выкладывай их в GitHub и копируй оба при переносе проекта на другой компьютер.
- Старые персонажи автоматически переводятся на новую систему классов при открытии; предметы, заклинания, заметки и значения характеристик не сбрасываются.
- Большой каталог использует открытый материал SRD 5.1 (2014) по лицензии CC BY 4.0; закрытые книжные описания не копируются.
- Полная атрибуция открытого материала находится в `ATTRIBUTION.md`.
